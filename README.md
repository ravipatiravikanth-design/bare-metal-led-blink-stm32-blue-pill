# 📖 Overview

This project implements LED blinking on the **STM32 Blue Pill (STM32F103CBT6)** using **bare-metal Embedded C** without relying on HAL, STM32CubeMX, or third-party libraries.

The firmware directly configures the **GPIOC** peripheral through memory-mapped registers to control the onboard **PC13 LED**. The project includes a custom startup file, linker script, and Makefile, providing a complete bare-metal firmware build and debug workflow.

## Build the Project

Open a terminal in the project directory and run:

```bash
make all
```

After a successful build, the following files are generated:

| File | Description |
|------|-------------|
| `blinkingled.o` | Object file generated from the application source |
| `startup.o` | Object file generated from the startup code |
| `blink.elf` | Final firmware executable |
| `blink.map` | Memory map generated by the linker |

---

## Flash the Firmware

Connect the **STM32 Blue Pill** to the **ST-Link V2** programmer and run:

```bash
make load
```

This starts OpenOCD using the ST-Link interface and the STM32F1 target configuration.

---

## Debug with GDB

Start GDB:

```bash
arm-none-eabi-gdb blink.elf
```

Connect to the target:

```gdb
target remote localhost:3333
```

Initialize the target:

```gdb
monitor reset init
```

Load the firmware:

```gdb
load
```

Run the application:

```gdb
monitor reset run
```

If you want to halt the processor after reset:

```gdb
monitor reset halt
```

Exit GDB:

```gdb
quit
```

# 🚀 Startup Sequence

After a system reset, the firmware follows the startup sequence implemented in **`startup.c`**.

### Vector Table

<p align="center">
  <img src="images/vector-table-1.png" width="850" alt="Vector Table">
</p>

The vector table is placed at the beginning of Flash memory and contains the initial stack pointer, reset handler, exception handlers, and interrupt handlers.

---

### Startup Flow

<p align="center">
  <img src="images/vector-table-2.png" width="850" alt="Startup Flow">
</p>

During startup, the firmware:

- Initializes the stack pointer
- Copies the `.data` section from Flash to SRAM
- Clears the `.bss` section
- Transfers control to `main()`

---

### Exception & Interrupt Handlers

<p align="center">
  <img src="images/vector-table-3.png" width="850" alt="Exception Handlers">
</p>

The startup file provides weak default handlers for exceptions and interrupts. These handlers can be overridden by user-defined implementations when required.

---

### Reset Handler

<p align="center">
  <img src="images/vector-table-4.png" width="850" alt="Reset Handler">
</p>

The `Reset_handler` is the application's entry point. It performs the required runtime initialization before calling the `main()` function, where the LED blinking application begins execution.
